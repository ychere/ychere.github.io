Track & Trace API v1.2

DCSA Compliant (TNT 1.1) Stable REST

Retrieve real-time status updates and event history for your containers and shipments using Booking Number, Bill of Lading Number, or Container Number.

Overview

The Track & Trace API provides visibility into the journey of your shipments. You can query the current status, location, and a detailed history of events associated with a specific container or booking. This allows for better planning, proactive exception management, and improved communication with stakeholders.

Key Features

  • Query by Booking Number, B/L Number, or Container Number.
  • Receive current shipment status (e.g., Gate Out, Loaded on Vessel, Discharged).
  • Access detailed event history with timestamps and locations.
  • Compliant with DCSA Track & Trace standards.
  • Supports standard RESTful principles.

Use Cases

  • Real-time shipment visibility dashboards.
  • Automated notifications for key shipment milestones.
  • Supply chain planning and optimization.
  • Customer service portals providing self-service tracking.

API Reference

GET /v1/track-trace/shipment-events

Retrieves shipment events based on provided query parameters.

Authentication

Requires OAuth 2.0 Bearer Token in the Authorization header.

Query Parameters

Parameter Type Required Description
bookingNumber String Conditional ONE Booking Reference. Required if B/L or Container # not provided.
billOfLadingNumber String Conditional Bill of Lading Number. Required if Booking or Container # not provided.
equipmentReference String Conditional Container Number (e.g., ONEU1234567). Required if Booking or B/L # not provided.
eventDateTime[gte] String (ISO 8601) Optional Filter events greater than or equal to this timestamp.

Error Responses

  • 400 Bad Request: Invalid or missing query parameters.
  • 401 Unauthorized: Invalid or missing authentication token.
  • 404 Not Found: Shipment not found for the provided identifiers.
  • 500 Internal Server Error: Unexpected server error.

Code Samples

Here are some examples of how to call the Track & Trace API.

cURL

curl -X GET \
'https://api.one-line.com/v1/track-trace/shipment-events?equipmentReference=ONEU1234567' \
-H 'Authorization: Bearer YOUR_ACCESS_TOKEN' \
-H 'Accept: application/json'

Python (requests)

import requests

api_url = "https://api.one-line.com/v1/track-trace/shipment-events"
access_token = "YOUR_ACCESS_TOKEN"
container_number = "ONEU1234567"

headers = {
    "Authorization": f"Bearer {access_token}",
    "Accept": "application/json"
}

params = {
    "equipmentReference": container_number
}

try:
    response = requests.get(api_url, headers=headers, params=params)
    response.raise_for_status() # Raise an exception for bad status codes (4xx or 5xx)

    data = response.json()
    print(data)

except requests.exceptions.RequestException as e:
    print(f"API Request Error: {e}")
except Exception as e:
    print(f"An error occurred: {e}")

JavaScript (fetch)

const apiUrl = 'https://api.one-line.com/v1/track-trace/shipment-events';
const accessToken = 'YOUR_ACCESS_TOKEN';
const containerNumber = 'ONEU1234567';

const headers = new Headers({
  'Authorization': `Bearer ${accessToken}`,
  'Accept': 'application/json'
});

const url = new URL(apiUrl);
url.searchParams.append('equipmentReference', containerNumber);

fetch(url, { method: 'GET', headers: headers })
  .then(response => {
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    return response.json();
  })
  .then(data => {
    console.log(data);
  })
  .catch(error => {
    console.error('Error fetching data:', error);
  });
